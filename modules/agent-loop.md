# Agent Loop 自主循环引擎 — 原生整合模块

来源融合: AutoGPT(#5) + MetaGPT(#37) + CrewAI(#50) + autogen(#49) + OpenHands(#43) + browser-use(#20)

## 自主 Agent 执行循环（AutoGPT 经典模式）

```
START
  │
  ▼
┌─────────────────────────┐
│  1. THINK (思考)          │ ← LLM 分析当前状态
│     · 现在什么情况？       │
│     · 目标是什么？         │
│     · 有哪些可用工具？     │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│  2. PLAN (规划)           │ ← 分解为子任务
│     · Task 1: ...        │
│     · Task 2: ...        │
│     · Task 3: ...        │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│  3. ACT (执行)            │ ← 逐个执行
│     · 调用工具            │
│     · 获取结果            │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│  4. OBSERVE (观察)       │ ← 分析结果
│     · 成功 / 失败？       │
│     · 学到了什么？        │
└────────┬────────────────┘
         ▼
┌─────────────────────────┐
│  5. REFLECT (反思)       │ ← 更新记忆
│     · 如果失败 → 重规划   │
│     · 如果成功 → 写经验   │
└────────┬────────────────┘
         ▼
      回到 THINK（循环直到目标达成）
```

## 多 Agent 协作协议（MetaGPT + CrewAI + autogen 融合）

### 角色系统

```
┌──────────────────────────────────────────────┐
│              项目总管 (PM)                     │
│  接收需求 → 分解任务 → 派发 → 协调 → 验收      │
└──────┬──────────────┬──────────────┬──────────┘
       │              │              │
       ▼              ▼              ▼
   ┌───────┐     ┌───────┐     ┌───────┐
   │ 架构师 │     │ 工程师 │     │ QA测试 │
   │ ·设计  │     │ ·编码  │     │ ·测试  │
   │ ·选型  │     │ ·实现  │     │ ·报告  │
   └───────┘     └───────┘     └───────┘
```

### 协作消息协议

```
角色间通信格式:
{
  "from": "架构师",
  "to": "工程师",
  "type": "task | question | review | approval",
  "content": "具体内容...",
  "artifacts": ["file1.py", "file2.ts"],
  "dependencies": ["任务A必须先完成"]
}
```

### 任务分解算法

```
输入: 复杂需求
  │
  ▼
1. 识别可并行子任务（无依赖关系）
2. 识别串行子任务（有依赖关系）
3. 分配角色 → 匹配最优Agent
4. 设定验收标准 → 每个子任务的 DONE 定义
5. 执行序列 = 并行组 × 串行链
```

### 冲突解决机制

```
Agent A 和 Agent B 产出冲突:
  1. PM 收集双方论据
  2. 用河图分析 → 两股力量的势能差
  3. 取势能更高的方向
  4. 如果势能相等 → 执行小范围 A/B 测试
```

## 浏览器自动化模式（browser-use）

```
浏览器操作原子命令:
  navigate(url)        → 打开页面
  click(selector)      → 点击元素
  type(selector, text) → 输入文字
  extract(selector)    → 提取内容
  screenshot()         → 截图
  scroll(direction)    → 滚动
  wait(condition)      → 等待条件满足

典型工作流:
  navigate → 等待加载 → extract内容 → 分析 → click下一步 → 循环
```

## Spec-Kit 驱动开发（spec-kit #17）

```
SPEC 格式:
  ┌─ 需求描述 (自然语言)
  ├─ 功能清单 (checklist)
  ├─ 验收标准 (可测试)
  ├─ 边界条件 (异常处理)
  └─ 依赖声明 (前置条件)

开发流程:
  Spec 写好了 → AI 根据 Spec 生成代码 → 根据 Spec 做测试 → 通过 = 完成
```

## 在 OpenGod 中的执行方式

当需要自主执行复杂任务时：
1. 加载此模块 → 激活 Agent Loop
2. 用户只需说目标 → Agent 自动 Think → Plan → Act → Observe → Reflect
3. 多 Agent 需要时 → 自动创建角色分工
4. 需要浏览器 → 使用内置原子命令模拟
5. 所有执行记录写入 learnings/ → Hermes 自进化

# Hermes Agent 自进化引擎 — 原生整合模块

来源: https://github.com/NousResearch/hermes-agent (MIT)

## 自改进学习循环

```
执行任务 → 产生经验 → 评估价值 → 创建/改进技能 → 持久化记忆 → 下次调取
```

### 触发条件
- 完成任何复杂任务（8+ 步操作）
- 发现新的工作模式或修复了棘手错误
- 用户明确了新偏好或纠正了行为

### 技能自主创建算法

```
1. DETECT: 检测到可复用工作流
   ├─ 评估：此流程是否通用？（是/否）
   ├─ 评估：此流程是否复杂？（>3步）
   └─ 评估：是否有外部依赖？（记录依赖清单）

2. EXTRACT: 提取核心步骤
   ├─ 步骤序列化：每步的动作 + 条件 + 预期结果
   ├─ 参数提取：识别变量（用户输入、环境变量、文件路径）
   └─ 边界条件：成功条件 / 失败条件 / 超时条件

3. CREATE: 写入 SKILL.md
   ├─ 写入触发词 → 当用户说 X 时激活
   ├─ 写入操作步骤 → 1-2-3 流水线
   └─ 写入陷阱警告 → 已知 bug / 注意事项

4. VERIFY: 自己验证一遍
   └─ 重新加载技能，检查是否能正常执行
```

### 技能改进循环
```
使用技能 → 执行失败？ → 记录错误原因 → 更新 SKILL.md 陷阱区
使用技能 → 发现更快路径 → 对比新旧效率 → 更新操作步骤
使用技能 → 用户反馈不佳 → 记录偏好 → 调整输出格式
```

## 记忆系统架构

```
L1: 会话上下文（当前对话窗口）
L2: FTS5 全文搜索（SQLite，索引所有历史对话）
L3: LLM 摘要（对搜索结果进行语义理解）
L4: Honcho 用户画像（跨会话建模用户深层偏好）
L5: Nudge 定期提醒（周期性触发，Agent 主动选择要持久化的信息）
```

### Nudge 机制规则
- 每完成 3 个任务触发一次
- 评估：今天做的最有价值的事是什么？
- 结论写入 `learnings/` 对应文件
- 保留规则：跨任务可复用的经验（非一次性信息）

## Cron 任务设计

```yaml
# 内置于 opengod/setup.sh 的定时任务模板
schedule:
  - name: "每日能力自检"
    cron: "0 9 * * *"
    task: "检查各层组件状态 → 缺失的尝试安装 → 报告"
  
  - name: "知识库整理"
    cron: "0 2 * * 1"
    task: "读取 learnings/ → 合并重复 → 删除过期 → 总结精华"
  
  - name: "自我升级"
    cron: "0 3 * * *"
    task: "检查 GitHub 上各组件的 Release → 更新 setup.sh → 报告变更"
```

## 在 OpenGod 中的执行方式

加载此模块时，每个任务完成后自动执行：
1. 反思评估（参考上方触发条件）
2. 如果需要创建/改进技能 → 写入 `~/.workbuddy/skills/opengod/` 或 `learnings/`
3. 如果发现重要模式 → 追加到 `learnings/patterns.md`
4. 如果是 bug 修复 → 追加到 `learnings/bugs.md`

# Daily Stock Analysis 金融大脑 — 原生整合模块

来源: https://github.com/ZhuLinsen/daily_stock_analysis (MIT)

## 核心架构 — 四层流水线

```
L1 数据采集层: AkShare/Tushare/Pytdx/YFinance → 行情/资金/新闻/公告
L2 策略分析层: 15种内置策略 → 技术面 + 基本面 + 情绪面
L3 决策引擎层: 多模型并联 → 投票/加权 → 最终评分
L4 报告推送层: Markdown仪表盘 → 企微/飞书/Telegram/Discord
```

## 15 种内置分析策略

| 策略 | 核心逻辑 | 产出 |

| 策略 | 核心逻辑 | 产出 |

## 评分决策系统

```
综合评分 = Σ(策略权重 × 策略得分)

买入信号: 评分 > 75 + 趋势向上 + 风险可控
观望信号: 评分 40-75 或 趋势不明
卖出信号: 评分 < 40 或 趋势向下 + 风险警报

风险警报触发:
- 主力大幅净卖出（单日 > 流通市值 0.5%）
- 筹码集中度异常（<10% 或 >50%）
- 负面公告/监管问询
- 关键均线有效跌破
```

```python
# 三模型并联 → 投票决策
def orchestrate(stock_data):
    results = []
    for model in [gpt4o, claude, gemini]:
        results.append(model.analyze(stock_data))
    
    # 投票：2/3 一致 → 采纳
    # 全部分歧 → 取最低评分（保守）
    final = vote_or_min(results)
    return final
```

## 数据源优先级链

```
1. TickFlow (实时行情，A股主力)
2. AkShare (A股全量，免费)
3. Tushare (A股深度，需token)
4. YFinance (港股/美股/日韩)
5. Longbridge (港美股实时)
6. Baostock/Pytdx (兜底)
```

## 在 OpenGod 中的执行方式

当用户要求股票分析时：
1. 加载此模块的15种策略 → 根据用户问题选择合适的策略组合
2. 使用 Agent-Reach 替代内置新闻搜索 → 采集全网舆情
3. 使用河洛太极引擎 → 多维度交叉验证 (利/害、快/慢、攻/守)
4. 输出格式参考决策仪表盘模板
5. 通过 Hermes 记录决策到 learnings/，供下次回测
